Responsive: the myth

What we've been told: Responsive is about media queries and things resizing

That's kind of true, but not the whole story...

So What is Responsive?

  • Prioritizing content
  • Adhering to Web Standards
  • Modern workflows and processes
  • Good UX practices

Responsive: Content is King

Content should be well-formed

Everything else should help to deliver content

Design and functionality no longer exist for their own sake

Web Standards: Good for Everybody

  • "Closed" tech like Flash, Silverlight, and Java only work on some devices, for some users, some of the time
  • The age of poor standards implementation in browsers is finally over
  • By making use of web standards (that fit your target devices), you make your work easier, more adaptable and more future-proof
  • When adhering to web standards, Good Things like accessibility and durability tend to come along "for free"

UX Practices

One Case: Graceful Degradation

  • Make intelligent use of fallbacks
  • Content will always delivered, even on old, oddball or otherwise non-targeted devices
  • All that should change are the non-critical design elements (the polish and sparkle)
  • Just one of many Responsive/"Mobile First" concepts
  • Closely related technique is Progressive Enhancement, which provides better experiences on prioritized devices
