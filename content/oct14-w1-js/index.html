The feature presentation: JavaScript!

A short history lesson

  • Made in 1995, during the "Browser Wars".
  • Originally LiveScript, became JavaScript because of a marketing decision.
  • Because of the intense competition during the Browser Wars, JavaScript was fully implemented in only ten days (whereas most other languages take many years).

So is it like Java?

  • Kinda. A lot of the syntax is taken from Java, but it acts totally differently.
  • It acts like two other languages, Scheme and Self. JavaScript takes its object-oriented nature from Self, and its functional nature from Scheme.
  • JavaScript relies heavily on functions, which are actually objects themselves. Java just has methods.

So what's all the fuss?

  • Loosely typed.
  • Object-oriented, but not class-based.
  • The only language native to the web.
  • Powerful Collections
  • Familiar ("Lisp in C's clothing").
  • Expressive.

Dynamic Typing

  • No need to worry about ints, doubles, Strings, or anything like that. Everything is simply a "var".
  • "Primitive types" are variables whose value is a number, string, boolean, undefined or null.
  • Everything else is an object.
  • The compiler will figure out typing for you (this often leads to entertaining results).

Native to the web

  • In the browser, all the HTML and CSS is turned into a hierarchical tree of objects (the DOM), which all have properties that JavaScript can access and modify and methods it can use.

Collections

  • JavaScript only has two collection types: Arrays and Objects.
  • Arrays are just like as in Java, but they are flexible, and don't need to be sized (they're actually just specialized objects)

Objects

  • Objects are simple name value pairs.
  • Objects can nest. This is incredibly powerful for representing data.
  • JSON has replaced XML as the data format of the web (and so much else).

Expressive

  • JavaScript gets things done with minimal complexity.
  • Few other programming languages can be used without being learned - the success of the modern web is proof of JavaScript being easy to use (for better or for worse!).
